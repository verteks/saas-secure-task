Аутентификация

Используется распространенный сценарий аутентификации по паролю. В качестве логина используется адрес электронной почты.

Безопасность

Пароли хранятся безопасно. Используются криптостойкие алгоритмы хэширования пароля с солью, что минимизирует опасность brute-force атаки и атаки с помощью радужных таблиц.

Функциональность

Реализованы следующие сценарии:

  • регистрация
  • аутентификация
  • выход
  • смена пароля
